## What You'll Learn

- SSL Monitor endpoints for certificate tracking

## SSL Monitor

The SSL Monitor extension tracks SSL certificate status across your child sites.

<Note>
Requires the [SSL Monitor Extension](https://mainwp.com/extension/ssl-monitor/) to be installed and active.
</Note>

---

### GET /ssl-monitor/info/

Retrieves SSL certificate information for all monitored sites.

**Method:** GET (readonly)

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://your-dashboard.com/wp-json/mainwp/v2/ssl-monitor/info/"
```

#### Response

```json
{
  "sites": [
    {
      "id": 1,
      "url": "https://example.com",
      "ssl_issuer": "Let's Encrypt",
      "ssl_valid_from": "2024-01-01",
      "ssl_valid_to": "2024-04-01",
      "ssl_days_remaining": 45,
      "ssl_status": "valid"
    }
  ]
}
```

---

### POST /ssl-monitor/check/

Triggers an SSL certificate check for all sites or a specific site.

**Method:** POST

```bash
curl -X POST \
  -H "Authorization: Bearer YOUR_API_KEY" \
  "https://your-dashboard.com/wp-json/mainwp/v2/ssl-monitor/check/"
```

#### Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `site_id` | integer | No | Check specific site only |

#### Response

```json
{
  "success": true,
  "message": "SSL check initiated for all sites"
}
```

---

### GET /ssl-monitor/{site}/

Retrieves SSL certificate information for a specific site.

**Method:** GET (readonly)

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://your-dashboard.com/wp-json/mainwp/v2/ssl-monitor/1/"
```

#### Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `site` | integer/string | Yes | Site ID or domain |

#### Response

```json
{
  "id": 1,
  "url": "https://example.com",
  "ssl_issuer": "Let's Encrypt",
  "ssl_valid_from": "2024-01-01",
  "ssl_valid_to": "2024-04-01",
  "ssl_days_remaining": 45,
  "ssl_status": "valid",
  "ssl_fingerprint": "AB:CD:EF:12:34:56:78:90"
}
```

## Pro Reports

The Pro Reports extension generates detailed activity reports for your child sites. All Pro Reports endpoints retrieve historical activity data within a specified date range.

<Note>
Requires the [Pro Reports Extension](https://mainwp.com/extension/pro-reports/) to be installed and active.
</Note>

### Common Parameters

All Pro Reports endpoints share these query parameters:

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `start_date` | string | Yes | Start date in Y-m-d format |
| `end_date` | string | Yes | End date in Y-m-d format |
| `action` | string | No | Filter by specific action type |
| `format` | string | No | Response format (default: normal) |

### Base Pattern

```
GET /wp-json/mainwp/v2/pro-reports/{site_id_or_domain}/{type}
```

---

### GET /pro-reports/{site}/plugins

Retrieves plugin activity for a site within the date range.

**Method:** GET (readonly)

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://your-dashboard.com/wp-json/mainwp/v2/pro-reports/1/plugins?start_date=2024-01-01&end_date=2024-01-31"
```

#### Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `site` | integer/string | Yes | Site ID or domain |
| `start_date` | string | Yes | Start date (Y-m-d) |
| `end_date` | string | Yes | End date (Y-m-d) |
| `action` | string | No | installed, activated, deactivated, updated, edited, deleted |

#### Response

```json
{
  "items": [
    {
      "name": "Yoast SEO",
      "action": "updated",
      "date": "2024-01-15",
      "old_version": "21.0",
      "new_version": "21.1"
    }
  ]
}
```
